  • //are used to make a short comment in javascript
  • /*This is a longer comment anything here will be ignored. This is a useful place to put notes*/
  • Javascript is made up from a series of statments-each one ends with a new line or semicolon. It is good practice to use both a new line and a semicolon.
  • A block is a series of statements that are collected togehter inside curly braces
    {const message = "hello";
    console.log(message); }
  • These are reserved words that are used by javascript and cant be used to name variables:
    abstract, await, boolean, break, byte, case, catch, char, class, const, continue, debugger, default, delete, do, double, else, enum, export, extends, false, final, finally, float, for, function, goto, if, implements, import, in instanceof, int, interface, let, long, native, new, null, package, private, protected, public, return, short, static, super, switch, synchronized, this, throw, throws, transient, true, try, typeof, var, volatile, void, while, with, yield
    These words should be reserved but aren't:
    undefined, NaN, Infinity
  • Primitive Data Types:
    1. String
    2. Symbol
    3. Number
    4. Boolean
    5. Undefined
    6. Null
  • Any value that isn't a primitive type like above is an object. These include arrays, functions, and object literals

  • A special operator typeof can be used to find out the type of value each is


  • Variables-used to refer to a value store in memory
